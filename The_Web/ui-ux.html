TOPIC: UI/UX

Behold, my contribution notes on User Interface/User Experience.

Nearest I can tell, the difference between UI and UX (thanks to this simple site) is the same as the difference between HTML and CSS. From an artistic standpoint, the UI (and CSS) work as the elements of art; lines, color, form, shading, etc. whereas UX (and HTML) function as the principles of art, or how everything is structured. Both need to be effective in order for the final product to work as well as it can, even though most often simplicity is the best direction to work towards. A UX can be anything as simple as a search field or entry field, while the UI is how that interface is presented. Site nav buttons also would classify as UX since they control where you end up, with the UI end being the button's appearance as something you are capable of interacting with. Once you understand it from that perspective, it's fairly simple to grasp while difficult to gleam how subtle and nuanced it can be.
