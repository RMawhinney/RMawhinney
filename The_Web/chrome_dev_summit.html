TOPIC: Chrome Dev Summit

Behold, old, my contribution notes on the Chrome Dev Summit 2016.

The video I watched was "Real Talk about HTTPS" wherein they discussed the worries, strategies and implements to handle HTTPS and its application to users. The current model idea was to incorporate a popup icon over non-HTTPS pages to warn the user that the site was not protected, but as mentioned in the video, an immediate implementation would mean that viewers would see it so commonplace that they'd learn to ignore it, completely counterproductive to warning about security issues!



The workaround for that would be to make sure HTTPS is the standard and unsecured HTTP is the rarity, to make sure when the warning appears it can be taken seriously. Unfortunately that would require most sites following an HTTPS protocol, and Chrome hopes, going forward, to streamline the transition with easy encryption services and advice for rate of transition. Really glad to see data security in general being taken so seriously.
