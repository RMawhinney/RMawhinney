HOW THE WEB WORKS

  • The Internet: massive network of millions of computers across the world
  • Email
  • SMS
  • App data
  • The web:

The World Wide Web: way of transmitting data over Internet using HTTP Protocol and HTML
Series of interactions between Clients and Servers



Clients: devices that request and render web content: browsers, mobile apps, screen readers, and content aggregators
Servers: applications that deliver content/service to clients. Important for programming or hosting



CLICKING A LINK/URL:
Uniform Resource Locator (URL) passed to Domain Name Server (DNS) which translates the URL into an IP address. Browser then uses IP address to locate Host Server and send request for content. Depending on filetype: For static sites: HTML text is sent to browser where it is rendered for user, while CSS/image requests sent and resources downloaded, and content rendered. Client-side content also rendered here For dynamic sites: Requests processed by web app (like PHP, .NET or Ruby), web server processes page and appropriate data is returned to browser to be rendered


DNS
Unique IPs to identify location and unique domain name for IPs. DNS=phone book of the internet. Hosting company usually associates these for you
Local ISP’s default DN
Root name servers: knows all name servers for “top level domains”
DNS caching: if you’ve visited recently, the DNS remembers IP on default


WHY DNS MATTERS
  If your site isn’t registered properly with a DNS, no one will be able to find you
  Multi-name sites need to point to the proper IP
  Make sure you’re aware of who site ownership is assigned to
  DNS caching can take up to 24 hours


Internet Protocols: simple languages that let computers share info with each other; The Internet Protocol Suite
Application Layer: HTTP, FTP, SMTP, DNS, RIP, SNMP
^Link Layer ^: Wireless Lan, Ethernet, ATM^
^Transport Layer: TCP, UDP^
^Internet Layer: IPv4, IPv6^


TCP/IP: Transmission Control Protocol and Internet Protocol: created 1973, main protocol suite used to exchange data over internet

HyperText Transfer Protol/HTTP: standard protocol for transferring data over WWW. “stateless protocol”, discarded after use: NOTHING about request remembered, like persistent data for browsing
-Can be handled through Cookies (text files written on client machine), scripting languages [JavaScript] Variables stored on server or server based Session Management.

COMMON WEB PROTOCOLS

  • FTP: File Transfer Protocol; enables transfer of large files [connects to File Server]
  • SMTP: Simple Mail Transfer Protocol; used to send mail [ISP for email;; in most clients, SMPT sends, Post Office Protocol/POP receives]
  • RTP: Real-Time Transfer Protocol; standard for audio/video [voice over/chat]
HTTPS (HTTP Secure): Layer of security protocol along with HTTP. Important to know when to discern

HTML is standard language for structuring content.
History: HTML 2.0 “standard: in ’95. 3.0 in ’97 had proprietary features. 4.0 in ’99 stabilized standards. XHTML 1.0 in ’00 hoped to replace HTML. Work on XHTML 2.0 abandoned in ’09. (’04)
Web Hypertext Application Technology Working Group (WHATWG) forms evolution of HTML resulting in publishing HTML5 draft, which is adopted by W3C(’08)


HTML uses tags to give elements meaning. Learning means learning syntax of language, which elements supported, and basic structure of HTML documents.
Simple HTML structure
 -Document type declaration (DTD)/Doc tag at top
 -opening and closing tags surround everything involved
 -Head of HTML contains all nonvisual elements(metatags with keywords or page descriptions, links to resources like the CSS) External script libraries can control presentation, layout, and functionality
 -After head tag is Body tag. All visual elements; headings, paragraphs, lists, quotes, images, links etc
 [tag 1][tag 2][/tag2][/tag1]


CSS Cascading Style Sheets: presentation language developed to control presentation of structured documents such as HTML files
Usually contained in an external file, with elements for a single page up to an entire site. Single file allows your entire site to follow a single style
External styles >Embedded styles >Inline styles. Styles can overwrite one another in case of conflict

Selector and Declaration. Selector tells browser which elements to style, and Declaration gives formatting instructions. All rules are set up as Property: values; with Properties separated by colon and semicolon to separate rules CSS BENEFITS

  • Allows for modular design
  • Faster development by centralization of design
  • Creates portable styles that can be reused or changed when necessary


CLIENT-SIDE SCRIPTING WITH JAVASCRIPT JAVASCRIPT: language for web design to add interactivity and application functionality. Created ’95 by Netscape to increase functionality of native browser, evolved into standard scripting language of the web
NO RELATION TO PROG LANG JAVA
-Allows designers to create interactive elements
-Can open new windows or give users feedback
-Can update, sort, and present data directly in browser
-No server interaction necessary
Suggestions for approaching Javascript
   Learn basics of native JavaScript first
   After that, experiment with popular Frameworks
   Have a strategy for providing fallback content for when JavaScript is disabled


COMMON IMAGE CONCERNS -To create web images, it’s best to understand how images work within browsers -Often need to find balance between filesize and quality -Especially important with the rise of mobile devices with higher res displays Use graphic file formats browsers support


Joint Photographic Experts Group (JPEG)

  • 1992 standard; among most widely used
  • High level of compression resulting in artifacts. JPEGs NOT recommended as source images

Graphics Interchange Format (GIF)
  • 8-bit graphics, meaning 256 colors
  • Support transparency and limited animations

Portable Network Graphics (PNG)
  • 1995; support transparencies as well with lossless compression
  • Supports millions of colors, albeit with larger filesize

Scalable Vector Graphics (SVG)
  • Vector graphics format, can be scaled independently of resolution
  • Written in SVG markup, making them editable via code and further styled in CSS. Widely supported


APIs (Application Programming Interfaces)
-set of instructions and standards for accessing services or components of a specific system
Open APIs=for other people to use
Mashups leverage multiple sites into a single app
API considerations
-Most APIs limit the amount of data or types of services they make available
-APIs evolve and change, make sure any service you’re using is monitored so that it continues to work as expected
-Have plan or fallbacks for when an API is not available


HTML5 APIs
Developed by WHATWG, then supported by W3C after chartering a working group
W3C took several APIs out of HTML5 and moved to their own specs, allowing APIs to mature independently and reduce the size of HTML5

  1. MEDIA API
    1. Integrated API designed to control loading and playback of V/A
    2. Related to Text Track API that is used to control captions and subtitles
  2. DRAG AND DROP API
    1. Allows you to enable drag and drop functionality on page elements
    2. Includes methods for handling drag and drop elements from outside the browser
  3. APP CACHE API
    1. Contains mechanisms for storing data offline
    2. Allows developers to create persistent applications that work even despite lack of internet
    3. Data can be synced when connections restored
  4. GEOLOCATION API
    1. Standalone API associated with HTML5
    2. Allows developers to find the current location of users and react to it
  5. CANVAS API
    1. Standalone associated with HTML5
    2. Gives ability to create a canvas area within the client that can be drawn or painted on
    3. Allows for the creation of complex games, animations, and other visual effects without the need for outside plugins


WEB FONTS
Technology that allows automatic downloading and temporarily installation of fonts within browser
Most frequently used to describe the CSS @fontface rule. Some early issues:
   Most font licenses included provisions which fontface would violate
  Font foundries and type designers didn’t like giving users the ability to download fonts
  Most formats at the time were very large, making implementation as a web resource difficult
  Font rendering needs onscreen are different from print, and most formats weren’t designed for onscreen usage

FONT FORMATS DESIGNED FOR WEBUSAGE
EOT Embedded OpenType
WOFF Web Open Font Format

WEB FONTS DEVELOPMENT
Font foundries and designers worked with browser developers to find solutions
In many cases they have redesigned/modified fonts for onscreen use, such as: reducing glyphs, creating screen-specific hinting, and new licenses which allow use through web fonts
OPTIONS
   -Font hosting services, like Typekit
   -You can also host the font yourself if you have the license


SERVER-SIDE SCRIPTING CGI scripts (Common Gateway Interface) Concerns:

  • Processing done on server, which increases server load for high traffic sites
  • Processing data usually requires a page refresh on the browser
  • AJAX and other client-side bridges help enhance usability of server-side scripting

Importance of PHP: DEVELOPMENT
  • 1994: Rasmus lerdorf writes a set of CGI scripts to track visitors on his resume: opens source to public
  • 1997: Andi Gutmans and Zeev Suraski rewrote original and released PHP 3.0, named Hypertext Preprocessor
  • Because it was free and open source it attracted a wide community of developers
  • Paired with opensource tools Linux, Apache, and MySQL, PHP became backbone of millions of sites

Available on almost all hosting planes
Large pool of existing developers to work with
PHP relatively easy to learn with lots of available resources

DATABASES: SUPER USEFUL
-Indexed DB and WebSQL
RELATIONAL DATABASES Store data in tables that relate to each other based on item key values
Allows for compelx sorting and filtering of data
Oracle, MySQL, Sybase and other SQL-based databases ae popular relational databases
NON-RELATIONAL DATABASES
Alternative to relational databases that are growing in popularity
NoSQL databases store data individual flat, object-driven datasets
These are often faster to index, and scale easier than relational databases
MongoDB, CouchDB, SimpleDB, CouchBase, and Cassandra are popular NoSQL databases


USING SQL DATABASES
SQL syntax is simple and logical, the basics are easy to learn
Select, insert, update, delets and other simple statements are used to retrieve data
Multiple versions of SQL with their own extensions
Queries usually written by other web app code
Dedicated apps like MySQL can help create and manage databases


CMS: Content Management System An application designed to control the creation, management, publishing and archiving of a site’s content CMS BENEFITS · Updating content can be done through a WYSIWYG editor, making it easy for non-technical people to update site content · Most CMS allow you to create user groups with various permission levels · This flexibility allows users to create content, pages, and regions within the site without involving a designer · Most CMS have plugins and extensions that add powerful features to sites EXPLORING CMS OPTIONS · Some CMS are hosted online and only require an account to start building hosted sites · Other CMS can be installed on your own web server · Features and page customization vary greatly between CMS, be sure to research the features you need · Popular CMS include Joomla, Drupal, Movable Type, ModX, Radiant, Wordpress, and Expression Engine


CDN: Content Delivery Network: a system for delivering content over a distributed network of servers
-Used to serve static resources such as javascript libraries, video, or other site dependencies
TYPES OF CDNs

  • Some CDNs host a single type of resource
  • CDNs like YouTube serve not only as delivery networks, but content aggregators as well
  • You can choose between numerous free or commercial CDN for dedicated service
  • Popular CDNs include Akamai, CloudFare, Amazon’s CloudFront, EdgeCast, and Windows Azure

ISSUES: Adds an additional layer of complexity
For dedicated resources, you don’t have the ability to create custom downloads or specific minified version
Certain security requirements might prevent the use of an external CDN
CLOUD COMPUTING
Distributed network of computers that can perform tasks by taking advantage of the scale and power of the network, Usually refers to distributed processes of the internet
GITHUB
Online distribution service that allows users to store repositories online that can be version controlled and shared by other users
Built around Git, an open source version control system
Allows you to manage projects, publish files, and store revisions
Offers both free and commercial plans
Features an easy to use web interface


JAVASCRIPT LIBRARYies extend the functionality of native Javascript. They ranged from focused sets like Date.js to the broader task based Jquery
PROS: Using libraries makes the process of writing Javascript more efficient, You can often extend your own coding capabilities through libraries, The code is widely used and tested with contributors improving the code on a consistent basis
CONS: Libraries increase the side of your code, and often add additional server requests, You can become tied to specific library syntax, Without knowing native Javascript you’re not always making an informed choice as to when a library is appropriate


FRAMEWORKS AND BOILERPLATES
Collection of prebuilt CSS, HTML, and JavaScript files designed to speed development of sites
BOILERPLATE: A set of templates built around a specific starting point/goal
Meant to provide starting point for building sites/applications
FRAMEWORKS: A collection of assets designed to help build sites or applications
Could include CSS grids, JavaScript libraries or helpful scripts, and HTML templates
COMPLEX UI FRAMEWORKS
Frameworks like Bootstrap are built around developing front end of sites or applications. These UI Frameworks function more like development tools. While they add the most overhead, they also give developers the widest set of tools to work with.


MINIMAL FRAMEWORKS
Typically have a singular goal or focus. Designed to have a small footprint and help out in specific areas The type of framework right for you will be largely based on project type
PROTOTYPING
Frameworks are often used to build HTML prototypes. Using a framework can dramatically speed up prototype development and iteration. In most cases framework code is NOT used in final product


CSS preprocessor
Scripting language that extends the functionality of CSS and must be compiled into native CSS code before publishing
-BENEFITS: additional functionality that CSS lacks, their syntax makes writing CSS more efficient, Many browsers inconsistencies are dealt with automatically when the CSS is processed
Sass: Written in Ruby, two distinct syntaxes
{Less}: Witten in JavaScript, syntax close to native CSS
Stylus: Written in JavaScript, minimal syntax
-CONS: In team environments, or if the production code is shared, everyone will need to know the preprocessor syntax
Preprocessed CSS can be bloated and inefficient
Additional layer of complexity to workflows
