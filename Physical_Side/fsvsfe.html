TOPIC: Full Stack vs Front-End

Behold, my scattered notes on Full Stack vs Front-End.

Becoming a Web Developer: Full Stack vs Front End
 -WD builds + maintains content for web targeting desktops, tablets, and handheld devices
-Front end work with HTML, CSS, and Java for in-browser content
-Back end develops for certain content customized per customer
-Full stack developers work with both, but both sides should understand both basically enough anyway with their own specialization


-HyperText Markup Language
 WD’s must understand information architecture: structural design of a website’s navigation
-Markup language defines roles of each piece of content and their navigation. Develops Document Object Model (DOM). Repairing/troubleshooting is incredibly essential to WD. All WDs must be masters of markup language
User Experience (UX) and User Interface (UI) specialization


CSS: Cascading Style Sheet, layout for websites. HTML is the principles of design, CSS is the elements. Best place to start is CSS core concepts Javascript: Language for managing in-browser interaction. DOM loading events, loading+unloading media, and creating+modding elements on page
Databases: [Foundations of Programming: Databases]
PHP and SQL stacks
Servers: devices sent content to browser when accessed. Back end builds/maintains, front end hancles designs/functionality WINDOWS + LINUX specific servers
Server-side scripting: working with code that runs on server. Main languages: .NET, PHP, Ruby, ASP.NET, NodeJS, Python. Start with PHP (marketable, widespread).
Most servers use variants of Relational DataBase Management System (RDBMS) called Sequel (SQL)
